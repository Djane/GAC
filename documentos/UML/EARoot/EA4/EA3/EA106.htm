Scenario Type Detail
0. Entrada de Dispositivos UpLoad Basic Path
<p>1. O Sistema apresenta no item "Dispositivos" do menu principal, o submenu "Carga de Dispositivos";<br/></p><p>2. O Ator clica nesse item;<br/></p><p>3. O Sistema executa o caso de uso "Upload e Informações de Pulseiras";<br/></p><p>4. O Ator confirma a carga;<br/></p><p>5. Caso de uso encerrado.<br/></p>
1. Entrada de Dispositivos via digitação Basic Path
<p>1. O Sistema apresenta no "Menu Principal", o item "Entrada de Dispositivo", no Menu "Dispositivo";<br/></p><p>2. O Ator digita as informações provenientes na página;<br/></p><p>3. O Siistema efetiva a gravação das informações no banco e muda o status do dispositivo para "Novo"; [A1]<br/></p><p>4. O caso de uso é encerrado.<br/></p>
2. [A1] Dispositivo existente na base Alternate
<p>1. O Sistema exibe uma mensagem de erro sobre a existência de um dispositivo idêntico na base de dados;<br/></p><p>2. O ator clica do botão "OK";<br/></p><p>3. O Sistema é desviado para o passo "2" do fluxo de "Entrada de Dispositivos via digitação".<br/></p>
3. Mudança de Status do Dispositivo Basic Path
<p>1. O sistema apresenta o item "Mudança de Status", no item de menu "Dispositivos", proveniente do "Menu Principal";<br/></p><p>2. O Ator escolhe o novo Status do Dispositivo, seguindo as opções possíveis e disponíveis;<br/></p><p>3. O Sistema apresenta um popup de confirmação;<br/></p><p>4. O Ator clica no botão "Sim"; [A2]<br/></p><p>5. O Sistema executa a mudança de Status e apresenta um popup de efetivação de tarefa;<br/></p><p>6. O Ator clica no botão "OK";<br/></p><p>7. O caso de uso é encerrado.<br/></p>
4. [A2] - O Ator clica no botão "Não" Basic Path
<p>1. O Sistema exibe um popup com a mensagem de desistência por parte do ator;<br/></p><p>2. O Ator clica no botão "OK";<br/></p><p>3. O fluxo é desviado para o passo "2" do "Mudança do Status do Dispositivo"<br/></p>
